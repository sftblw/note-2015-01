### 4.2.2 디지털 사운드로의 변환

-	과정 : 표본화 → 양자화 → 부호화

#### (1) 표본화

-	많이 뽑으면 좋음
-	1초동안에 취한 표본 수 (Hz)
-	나이키스트 정리 (Nyquist Theorem)
	-	> 표본화를 할 때 원음을 그대로 반영하기 위해서는 원음이 가지는 **최고 주파수의 2배 이상으로 표본화**해야 한다.
	-	최고주파수의 (최소) 2배 이상으로 표본화해야
	-	→ CD가 44.1KHz 인 이유 : 가청주파수가 20KHz 정도라

#### (2) 양자화

-	표현의 정밀함의 정도 : 화면에도 해상도가 있듯이.
-	음의 해상도 : 표본화하는 정밀도
-	8bit 양자화 → 256단계, 음악은 16bit → 65536 단계

4.3 사운드의 저장과 고품질화
----------------------------

### 4.3.1 디지털 사운드 파형의 저장 방식

-	용량을 줄여야.

#### (1) PCM 방법 (Pulse Coded Modulation)

-	입력값을 그대로 기록 → 용량이 크다
-	Pulse? → 파형!
-	\* 모뎀 (Modem)
	-	→ 사운드 (소리) 에 대한 변조 + 복조 기기 (SW)
		-	(Modulation + Demodulation)
-	옛날 집에서 썼던 방식

#### (2) ADPCM 방법 (Adaptive Differential ~) : 적응적 차분 코딩

-	IMA (International Multimedia Association) 에서 제안
-	표본화된 신호의 차이를 저장하는 방식
	-	차이만 저장 : DPCM
	-	DPCM은 인접한 값과의 차이가 크면 비효율적
-	ADPCM
	-	> DPCM에서 인접한 값과의 차이가 크면 진폭을 나누는 단계를 크게 하고, 차이가 작으면 진폭을 나누는 단계를 작게 하여 가변적으로 차이를 정밀하게 저장

#### 압축 알고리즘 정리

-	예측 코딩 (Predictive Coding)
	-	차이 PCM (DPCM)
	-	적응적 차이 PCM (ADPCM)
	-	델타 변조 (DM : Delta Modulation)
	-	적응적 DM (ADM)
-	DPCM
	-	기울기 문제 : 갑자기 엄청 큰 소리가 나왔을 때 → 차이 저장?

#### DPCM과 ADPCM의 예

-	-12를 +8로 바꾸세요
-	5비트 : 32 : -16~15 : 소리가 달라짐 (!!!!)
-	ADPCM 의 경우 스케일 변화 정보를 곱해서 더함 → 덜 왜곡

#### 파일 크기

-	표본화율 x 해상도 x 모드 (mono = 1, stereo = 2) x 시간 (초)
